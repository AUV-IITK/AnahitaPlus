\hypertarget{md_hardware_layer_hardware_imu_README_autotoc_md61}{}\doxysection{Overview}\label{md_hardware_layer_hardware_imu_README_autotoc_md61}
This is a R\+OS package for interfacing with the \href{http://px4.io/}{\texttt{ P\+X4}} hardware or the \href{https://www.spartonnavex.com/product/ahrs-8/}{\texttt{ Sparton A\+H\+R\+S-\/8}} hardware.

For the P\+X4, we are using the in-\/built inertial measurement unit (I\+MU) present in the P\+X4 for the robot. For the Sparton A\+H\+R\+S8, the driver communicates with the sensor using N\+M\+EA protocol and publishes the I\+MU data as R\+OS sensor messages.

The {\ttfamily hardware\+\_\+imu} package has been tested under \href{http://www.ros.org}{\texttt{ R\+OS}} Kinetic and Ubuntu 16.\+04 L\+TS. This node takes the I\+MU data in quateranion format and convert it into euler angles. We calculate the roll, pitch and yaw using the following formulas\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{yaw = atan2 ( 2 * x * y -\/ 2 * w * z, 2 * w * w + 2 * x * x -\/ 1)}
\DoxyCodeLine{roll = atan2(2 * q2 * q3 -\/ 2 * q0 * q1, 2 * q0 * q0 + 2 * q3 * q3 -\/ 1)}
\DoxyCodeLine{pitch = -\/asin(2 * q1 * q3 + 2 * q0 * q2)}
\end{DoxyCode}
