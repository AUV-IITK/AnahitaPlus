\hypertarget{md_hardware_layer_hardware_arduino_README_autotoc_md29}{}\doxysection{Overview}\label{md_hardware_layer_hardware_arduino_README_autotoc_md29}
This is a R\+OS package for interacting arduino through the \href{http://wiki.ros.org/rosserial_arduino}{\texttt{ rosserial\+\_\+arduino}} package. The package is meant to actuate thrusters connected to the arduino through a motor driver, and publish the measurements taken from various sensors connected to the arduino onto separate topics.

The {\ttfamily hardware\+\_\+arduino} package has been tested under \href{http://www.ros.org}{\texttt{ R\+OS}} Kinetic and Ubuntu 16.\+04 L\+TS. The source code is released under a B\+SD 3-\/\+Clause license.

The hardware used are as follows\+:

{\bfseries{N\+O\+TE\+:}} This connects the arduino always to the port named {\ttfamily /dev/arduino}.\hypertarget{md_hardware_layer_hardware_arduino_README_autotoc_md33}{}\doxysubsection{Preparing the Serial Port}\label{md_hardware_layer_hardware_arduino_README_autotoc_md33}
Arduino will likely connect to computer as port {\ttfamily /dev/arduino}. The easiest way to make the determination is to unplug all other U\+SB devices, plug in your Arduino, then run the command\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{ls /dev*}
\end{DoxyCode}


Next make sure you have read/write access to the port. Assuming your Arduino is connected on {\ttfamily /dev/arduino}, run the command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{ls -\/l /dev/arduino}
\end{DoxyCode}


You should see an output similar to the following\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{crw-\/rw-\/-\/-\/-\/ 1 root dialout 166, 0 2018-\/03-\/05 08:31 /dev/arduino}
\end{DoxyCode}


Note that only root and the \char`\"{}dialout\char`\"{} group have read/write access. Therefore, you need to be a member of the dialout group. You only have to do this once and it should then work for all U\+SB devices you plug in later on.

To add yourself to the dialout group, run the command\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo usermod -\/a -\/G dialout your\_user\_name}
\end{DoxyCode}


where {\ttfamily your\+\_\+user\+\_\+name} is your Linux login name. You will likely have to log out of your X-\/window session then log in again, or simply reboot your machine if you want to be sure.

When you log back in again, try the command\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{groups}
\end{DoxyCode}


and you should see a list of groups you belong to including dialout.\hypertarget{md_hardware_layer_hardware_arduino_README_autotoc_md34}{}\doxysubsection{Building Arduino code}\label{md_hardware_layer_hardware_arduino_README_autotoc_md34}
